                                                                                                                                                                         1. **Unit Tests**\n    *   **Description:** These tests focus on verifying the individual components of the script in isolation. This includes testing the `ServiceManager` class's methods for starting, stopping, and monitoring services, as well as the `respawn_self` function's logic.\n    *   **Methodology:** Use a testing framework like `pytest` or `unittest`. Mock external dependencies such as `subprocess.Popen`, `os.killpg`, and `signal` to isolate the logic being tested. This allows for controlled simulation of service failures, restarts, and shutdown signals.\n    *   **Example Test Case (Conceptual):**\n        *   **Test:** Verify `ServiceManager.monitor_services` correctly identifies a terminated process.\n        *   **Setup:** Create a mock `ServiceManager` instance with a simulated running process.\n        *   **Action:** Mock the `process.poll()` method to return a non-None exit code.\n        *   **Assertion:** Assert that the process is removed from `self.processes` if the restart policy does not allow for restart, or that `start_service` is called if the policy dictates a restart.\n\n**2. Integration Tests**\n    *   **Description:** These tests validate the seamless interaction between the different components of the script, particularly the self-loading mechanism and the service management functionalities. They ensure that the script behaves as expected when its components work together.\n    *   **Methodology:** Run the `integrated_script.py` script in a controlled environment. Use `subprocess` to launch the script and then interact with it by sending signals (SIGINT, SIGTERM) or by manually terminating the child processes it spawns. Monitor the script's output logs and resource usage.\n    *   **Key Scenarios:**\n        *   **Successful Service Lifecycle:** Start the script and confirm all defined services (`example_service_1`, `example_service_2`, `failing_service_demo`) are running.\n        *   **`on-failure` Respawn:** Terminate `failing_service_demo`. Observe logs to confirm it is restarted up to `max_retries` (3 times) with the specified delay (3s).\n        *   **`always` Respawn:** Terminate `example_service_1`. Observe logs to confirm it is restarted immediately.\n        *   **Max Retries Reached:** Ensure `failing_service_demo` stops being restarted after exceeding its `max_retries`.\n        *   **Graceful Shutdown:** Send SIGINT (Ctrl+C) to the main script.
